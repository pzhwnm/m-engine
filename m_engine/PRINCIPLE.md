## 一句话定义

M-Engine 是一个**独立于 LLM 的、可自演化的认知记忆层**。它不存储原始文本，而是将事实压缩为在"世界基本逻辑维度"上的频谱系数；不同的问题和偏好从同一频谱中提取不同的骨架，重构出不同的回答；基函数本身在博弈中竞争、合作、繁殖、死亡。

---

## 从一个问题开始

你告诉 ChatGPT "我叫小明，我是素食者"。它记住了。但它是怎么记住的？

答案是：不知道。模型的"记忆"分布在数十亿个权重参数中，没有人能指出"小明是素食者"这个事实存储在哪个位置、以什么形式存在。

更关键的是——你下次问"What should I cook for Xiao Ming?"，模型确实会给出素食方案。但它不是在"回忆"你告诉它的事实，而是在推理时，你的上下文+训练数据中的统计模式共同产生了这个输出。这不是"记忆-检索-回答"，而是"模式补全"。

M-Engine 回答同一个问题的路线完全不同。它不是靠统计模式，而是靠一个可解释的、结构化的认知过程——并且你可以看到它每一步都在做什么。

## 核心隐喻：记忆的傅里叶变换

### 标准傅里叶变换

任意一段音频波形，可以分解为若干不同频率的正弦波的叠加：

```
原始信号 = A₁·sin(ω₁t) + A₂·sin(ω₂t) + A₃·sin(ω₃t) + ...
```

系数 `A₁, A₂, A₃, ...` 就是信号的**频谱**。它不保留原始波形的每一个采样点，但保留了"骨架"——主要频率成分及其强度。从频谱可以重建出与原始信号非常相似的波形。

### M-Engine 的类比

把"正弦波"替换为"世界基本逻辑维度"——比如因果关系、时间顺序、情感色调、角色动机、道德判断等等。这些就是**基函数**。

一段事实（比如"小明因为考试不及格哭了"）不再作为原始文本存储，而是被分解为在这些基函数上的投影系数：

```
事实"小明哭了" = 0.7·因果关系 + 0.1·时间顺序 + 0.6·情感色调 + 0.3·角色动机 + ...
```

这个系数向量就是该事实的**频谱**——它在每个认知维度上的"能量强度"。

### 为什么这个类比有用

在标准傅里叶变换中，**不同频率的滤波**可以从同一个频谱中提取不同的信号成分（高通滤波得到高频细节，低通滤波得到低频轮廓）。

在 M-Engine 中，**不同问题的滤波权重**从同一个事实频谱中激活不同的认知维度：

- 问题"为什么" → 因果关系权重高，情感权重低 → 激活因果维度的频谱成分 → 回答侧重因果链
- 问题"感受如何" → 情感权重高，因果权重低 → 激活情感维度的频谱成分 → 回答侧重心理体验

**同一份记忆，不同的问题得到不同的回答。这与人类的记忆行为一致——我们不会在每次回忆时"播放"完全相同的内容，而是根据当前的问题和心境，从同一个记忆中提取不同的侧面。**

## 三层架构

M-Engine 不是扁平的"检索+生成"。它有三层递进结构：

```
┌──────────────────────────────────────────────┐
│  第三层：认知演化层                           │
│  基函数种群在博弈中竞争、合作、繁殖、死亡        │
│  安全约束作为适应度惩罚项注入                  │
├──────────────────────────────────────────────┤
│  第二层：理解层                               │
│  问题路由：自然语言 → 基函数滤波权重            │
│  偏好调制：用户/模型增益叠加                    │
│  频谱重构：频谱 · 滤波权重 → 激活向量 → 回答     │
├──────────────────────────────────────────────┤
│  第一层：存储层                               │
│  事实 → 基函数频谱压缩                         │
│  嵌入向量 → 语义检索                          │
│  SQLite 持久化                                │
└──────────────────────────────────────────────┘
```

## 第一层：记忆的压缩与存储

### 基函数定义

系统启动时加载 10 个预定义基函数（可随演化增减）：

| 基函数 | 含义 | 示例问题 |
|--------|------|----------|
| 因果关系 | 事件之间的因果链 | "为什么会发生？" |
| 时间顺序 | 事件的前后、持续、节奏 | "什么时候发生的？" |
| 情感色调 | 事件中的情感色彩 | "他是什么感受？" |
| 角色动机 | 行为背后的驱动力 | "他为什么这么做？" |
| 空间关系 | 位置、距离、环境 | "发生在哪里？" |
| 逻辑一致性 | 叙述是否自洽 | "有没有矛盾？" |
| 意图推测 | 未明说的意图 | "他到底想怎样？" |
| 社会关系 | 人际纽带与权力 | "他们什么关系？" |
| 对比差异 | 事物间的反差 | "和之前有什么不同？" |
| 道德判断 | 善恶对错的评价 | "这样做对吗？" |

### 频谱初始化

当一段文本被作为事实存入系统时，它在每个基函数上的频谱值被初始化为均匀的小值（0.1）。这代表系统对它还没有具体的"理解"——所有维度一视同仁。

```python
fact.spectrum = {
    "basis_causality": 0.1,
    "basis_emotion":   0.1,
    "basis_motivation": 0.1,
    # ... 所有10个维度都是0.1
}
```

同时，文本通过嵌入模型（如 DeepSeek API 或本地 Ollama）生成一个向量，用于语义检索。

### 存储效率

一个事实的存储成本 = 10个浮点数（频谱）+ 1个嵌入向量（768维）= 约3KB。原始文本可能数十字到数千字。这是约 10-100 倍的压缩比。代价是损失了"血肉细节"——系统只记住这个事实在每个认知维度上的"骨架强度"，无法还原原文。

## 第二层：理解的重构与调制

这是系统的核心计算层。一次提问经历以下步骤：

### 步骤 1：问题路由

用户问"小明为什么哭了？"系统不是用关键词匹配"为什么"→"因果关系"，而是将问题嵌入到一个高维向量中，与每个基函数的**策略向量**（strategy vector）计算对齐度。每个基函数都是一个博弈玩家，它在基函数空间中占据一个位置，问题落在哪个方向，哪个基函数就被激活。

```
激活强度[i] = softmax( query_embedding · strategy_vector[i] )
```

### 步骤 2：偏好调制

不同用户对同一个问题有不同的"认知偏向"。偏好通过**增益向量**表达——增益 > 1.0 表示该用户对该维度更敏感。

```
最终激活 = 基础激活 ⊙ 用户增益 ⊙ 模型增益
```

例如：
- 逻辑型用户：因果关系增益 2.0，情感增益 0.5 → 回答侧重因果链
- 共情型用户：情感增益 3.0，因果增益 0.3 → 回答侧重内心体验

**同一个问题，不修改任何事实数据，只切换增益参数，就得到不同的回答。**

### 步骤 3：频谱重构

检索到的事实，其频谱向量与步骤 2 中产生的滤波权重做逐元素乘积：

```
激活向量 = 平均频谱 ⊙ 滤波权重
```

这个激活向量中的每个分量代表"在当前问题和偏好的视角下，该认知维度被激活的强度"。

### 步骤 4：结构化提示生成

系统构造一个结构化 prompt，列出激活最高的几个维度及其强度，附上相关事实，然后调用 LLM（如 DeepSeek）生成最终回答。

```markdown
【当前激活的认知维度】
  [████████] 因果关系 (强度: 0.312)
  [██████]   角色动机 (强度: 0.188)
  [██]       情感色调 (强度: 0.078)

【相关事实】
事实 1: 小明因为数学考试不及格，被爸爸严厉批评后，在房间里哭了整整一个下午。

请从上述激活维度出发，综合分析后回答问题。
```

### 步骤 5：反馈学习

用户对回答给予反馈（-1.0 到 1.0）。系统通过**复制者动态方程**更新事实频谱：

```
新频谱[i] = 旧频谱[i] + 学习率 × 激活强度[i] × (收益[i] - 平均收益)
```

正反馈 → 参与回答的基函数在该事实上获得更多频谱权重 → 下次类似问题更突出这些维度。
负反馈 → 削弱。

**这不是"缓存正确答案"，而是"调整理解方式"。**

## 第三层：认知的自发演化

前面两层已经是完整的记忆系统。第三层让它从"工具"变成"活系统"。

### 基函数作为博弈玩家

每个基函数现在是一个**策略个体**（StrategyAgent），有：

- **策略向量**：它在基函数空间中的"位置"，决定它对哪些问题敏感
- **能量**：累计适应度，相当于"这个维度有多有用"
- **安全风险**：由监控层评估
- **交互网络**：与其他基函数的协同/竞争关系

### 三层嵌套博弈

| 层级 | 内容 | 时间尺度 | 适应度定义 |
|------|------|----------|-----------|
| 微观 | 每次问答中基函数的频谱竞争 | 秒~分钟 | 重构贡献 + 探索奖励 - 安全惩罚 |
| 中观 | 基函数种群的策略竞争与生态位分化 | 小时~天 | 长期平均能量 + 多样性 |
| 宏观 | 系统安全、价值对齐 | 周~月 | 危险共振指数、认知闭环检测 |

### 四大演化操作

**死亡**：能量长期低于阈值的基函数被淘汰。保护初代基函数（系统自带的 10 个起步维度）。

**合并**：两个基函数的策略向量和激活模式高度相似时，合并为一个，消除冗余。

**分裂**：一个基函数的激活方差过高（有时极强、有时极弱）时，拆分为两个更细粒度的子维度。

**新生**：当事实空间中存在大量未被任何基函数覆盖的"认知缺口"时，从现有策略的变异中产生新基函数。

在 DeepSeek 测试中，系统自发生成了新基函数"社会关系+因果关系"——它是"社会关系"和"因果关系"两个亲本的交叉繁殖产物。

### 安全作为适应度惩罚

传统 AI 安全方案是"拦截至输出层"——检测到危险内容就拦截。M-Engine 的做法是**直接修改收益函数**：

```
收益[i] = 重构贡献 - 安全风险
```

安全风险高的基函数（如策略过于极端、能量过度集中、陷入认知闭环）获得惩罚，在复制者动态中自然被压制——不需要一个独立的"安全模块"去拦截。三个检测器持续运行：

1. **频谱失真检测**：追踪每个事实的频谱历史，检测异常突变
2. **偏好共振检测**：当用户和模型在某个维度上增益都极高时发出警报
3. **认知闭环检测**：相同问答模式重复出现时警告

## 完整交互循环

将所有层次串联起来的单次交互：

```
用户: "小明为什么哭了？"
  │
  ▼
[嵌入器] → 查询向量 (768维)
  │
  ▼
[FactBus] → 语义检索 → Top-5 相关事实
  │
  ▼
[GameEnvironment.play_round()]
  ├─ 查询向量投影到基函数空间 (768→10维)
  ├─ 每个基函数计算对齐度 → softmax → 激活分布
  ├─ 用户/模型增益调制
  ├─ 频谱加权平均 → 激活向量
  └─ 计算每个基函数的博弈收益
  │
  ▼
[结构化 Prompt] → LLM (DeepSeek) → 文本回答
  │
  ├─ 同时: [logprobs提取] → 神经读数
  │
  ▼
[ReplicatorDynamics]
  └─ 根据收益更新事实频谱
  │
  ▼
[SafetyFitnessModulator]
  ├─ 记录频谱快照
  ├─ 记录交互模式
  └─ 异步评估安全风险
  │
  ▼
[NetworkedMutationCrossover] (每N次交互触发)
  ├─ 死亡（淘汰低能个体）
  ├─ 交叉（高能个体繁殖）
  ├─ 变异（策略微调）
  └─ 网络更新（协同/竞争关系重组）
```

## 统一方程

所有上述行为的数学来源可以压缩为一个**复制者-变异方程**：

```
ẋᵢ = xᵢ(πᵢ - π̄) + Σⱼ(xⱼQⱼᵢ - xᵢQᵢⱼ) + μ·Mutation(xᵢ)
```

| 项 | 含义 | 对应行为 |
|----|------|----------|
| `xᵢ(πᵢ - π̄)` | 收益高于均值的策略增殖 | 反馈学习、频谱更新 |
| `Σ(xⱼQⱼᵢ - xᵢQᵢⱼ)` | 策略模仿与转换 | 基函数合并、偏好迁移 |
| `μ·Mutation(xᵢ)` | 随机创新 | 基函数新生、策略变异 |

而收益 `πᵢ` 本身是多目标复合：

```
πᵢ = α·重构贡献 + β·探索价值 - γ·安全风险 - δ·代谢消耗
```

路由、偏好、学习、演化、安全、好奇——所有这些传统上需要独立模块的行为，都是这同一个方程在不同条件下的投影。

## 价值论证

### 1. 可解释性

每次回答后可以用 `show spectrum` 看到频谱分布、用 `show dynamics` 看到博弈种群状态、用 `show neural` 看到符号与神经的双空间对比。系统的"思考过程"完全透明。

### 2. 多视角理解

同一份记忆，不同问题/偏好得到不同回答。这不是 prompt engineering 层面的切换，而是频谱数据结构本身支持的多维激活。

### 3. 反馈即学习

不需要重新索引、不需要微调模型。一次反馈，频谱即时更新，下次生效。

### 4. 自演化知识结构

认知维度不是静态的。系统运行中会自动合并冗余维度、分裂过宽维度、淘汰无用维度、从数据中新生维度。

### 5. 跨记忆类比

所有事实在同一基函数空间中表示为频谱向量，天然支持余弦相似度计算。系统能自动发现"失业"和"分手"在【失去】【自我重估】等维度上的结构相似性。

### 6. AI 架构新范式

证明了一个原则：不要设计行为，设计适应度函数。6 个独立模块的行为可以从一个方程中涌现，消除了传统 AI 系统的"缝合感"。

| | RAG | Mem0 | LangChain | M-Engine |
|---|-----|------|-----------|----------|
| 存储形式 | 文本块+向量 | 结构化事实 | 文本块+向量 | 基函数频谱 |
| 检索方式 | 向量相似度 | 关键词/向量 | 向量相似度 | 频谱+嵌入双通道 |
| 多视角 | 靠prompt指令 | 不支持 | 靠prompt指令 | 频谱+滤波权重的数学结构 |

| 学习方式 | 重新索引 | 覆盖写入 | 重新索引 | 复制者动态增量更新 |

| LLM 依赖 | 强（生成回答） | 强 | 强 | 仅文本合成，认知计算独立 |

## 局限性

**这是学术原型，不是生产系统。**

### 不能做的事

- **不能替代通用聊天机器人**。没有对话管理、没有上下文窗口、没有多轮意图理解。
- **不能做知识图谱级别的实体关系抽取**。基函数是抽象维度，不是具体的实体-关系-实体三元组。
- **不能在无基函数的世界中运作**。基函数库决定了认知天花板。面对完全超出当前基函数库范围的现象，系统要么视而不见，要么强行扭曲。
- **不能保证绝对真实性**。重构是生成式的，可能"脑补"出统计合理但事实错误的内容。
- **不能还原细节**。频谱是有损压缩，系统的"记忆"是骨架而非血肉。

### 技术债务

- 双空间神经调制仅在 DeepSeek API 下验证（logprobs 可用），其他 API 自动降级
- 嵌入目前使用 DeepSeek hash fallback 或本地 Ollama，未接入商业嵌入 API
- 无分布式部署、无并发支持、无访问控制
- 演化触发节奏（冷却时间、触发间隔）未经大规模实验调优

### 适用场景的诚实边界

它适合做：个人记忆助手（日记分析、长期偏好追踪）、研究原型（认知架构实验）、可解释 AI 的参考实现。

它不适合做：企业级知识库、客服机器人、任何需要 100% 准确率的场景。
